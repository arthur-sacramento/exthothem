QuickGuide

Overview
The goal of the system is to build ready HTML pages that can be easily copied or exported. When using a traditional database like MySQL, several intermediate steps are performed, which, while providing flexibility, sometimes increases complexity or requires more resources

1. Database

Instead of using a large file as a database, data will be stored in numerous small documents such as cars_1.html, cars_2.html, cars_3.html and so on, serving both as a category ready to be viewed or searched, as well as a navigation to the next page result. When the limit is exceeded (of current file), a new document will be created for store the new data.

2. Search

Synonyms and category detection can be used into the search system. For example, if users search for 'book' or 'pdf', they will be redirected to the table or document 'book.html'. Similarly, if the search query matches a category, users will be redirected accordingly. For instance, a search for 'beautiful art' will redirect to the table 'art', that is, any sentence containing the word 'art' will direct to the 'art' category or table.

3. Custom Styling

Users have the option to choose a custom style for their posts. By default, there are 'default' and 'simple' options available.

4. Others

There are other options and tools in menu_contents.php
